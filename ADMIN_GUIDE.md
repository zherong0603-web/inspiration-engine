# 管理员使用指南

## 📌 如何成为管理员

### 方法 1：使用演示账号（推荐）
直接使用演示账号登录，自动拥有管理员权限：
- **邮箱**：demo@example.com
- **密码**：demo123456

### 方法 2：第一个注册的用户
系统中第一个注册的用户自动成为管理员。

### 方法 3：手动设置（开发者）
如果需要给其他用户添加管理员权限，可以修改数据库：

```bash
# 进入数据库
sqlite3 dev.db

# 查看所有用户
SELECT id, email, name, createdAt FROM User;

# 记住要设置为管理员的用户 ID
# 然后修改代码 app/api/auth/is-admin/route.ts
# 添加该用户的邮箱或 ID 到管理员列表
```

## 🎯 如何查看用户反馈

### 步骤 1：登录管理员账号
使用管理员账号登录系统。

### 步骤 2：进入管理后台
点击右上角用户头像 → 选择「⚙️ 管理后台」

### 步骤 3：查看反馈列表
1. 在管理后台页面，点击「💬 用户反馈」标签
2. 可以看到所有用户提交的反馈，包括：
   - 反馈类型（Bug/建议/其他）
   - 反馈状态（待处理/处理中/已解决/已关闭）
   - 反馈标题和内容
   - 提交时间
   - 用户信息（邮箱）
   - 联系方式（如果用户填写了）

### 步骤 4：查看数据统计
点击「📊 数据统计」标签，可以看到：
- 总用户数
- 活跃用户数（最近 7 天）
- AI 调用次数
- 平均响应时间
- 内容总数、选题总数、反馈总数
- 最受欢迎的功能排行

## 📊 管理后台功能详解

### 1. 用户反馈管理

**查看反馈详情：**
- 反馈类型：了解是 Bug 报告还是功能建议
- 反馈内容：查看用户的详细描述
- 用户信息：知道是谁提交的反馈
- 联系方式：方便跟进和回复

**反馈状态说明：**
- 🟡 待处理：新提交的反馈
- 🔵 处理中：正在处理的反馈
- 🟢 已解决：问题已修复或建议已采纳
- ⚫ 已关闭：不需要处理的反馈

**处理反馈的建议流程：**
1. 定期查看反馈列表（建议每天 1-2 次）
2. 优先处理 Bug 报告
3. 评估功能建议的可行性
4. 如果用户留了联系方式，及时回复
5. 记录常见问题，更新帮助文档

### 2. 数据统计分析

**用户活跃度：**
- 总用户数：注册用户总量
- 活跃用户：最近 7 天有登录的用户
- 活跃率 = 活跃用户 / 总用户数

**功能使用情况：**
- AI 调用次数：了解 AI 功能的使用频率
- 平均响应时间：监控 AI 性能
- 功能排行：知道哪些功能最受欢迎

**内容数据：**
- 内容总数：用户创建的内容数量
- 选题总数：用户保存的选题数量
- 反馈总数：收到的反馈数量

## 🔍 常见问题

### Q1：为什么我看不到管理后台入口？
**A：** 可能原因：
1. 你不是管理员账号
2. 需要刷新页面
3. 检查是否正确登录

**解决方法：**
- 使用 demo@example.com 账号登录
- 或者使用第一个注册的账号
- 刷新页面后重新查看

### Q2：反馈列表是空的？
**A：** 可能原因：
1. 确实没有用户提交反馈
2. API 请求失败

**解决方法：**
- 先自己提交一条测试反馈
- 打开浏览器控制台查看是否有错误
- 检查网络连接

### Q3：如何回复用户反馈？
**A：** 当前版本暂不支持在系统内直接回复，建议：
1. 如果用户留了联系方式，通过邮箱或微信回复
2. 后续版本会添加回复功能

### Q4：如何导出反馈数据？
**A：** 当前版本需要手动导出：
```bash
# 进入数据库
sqlite3 dev.db

# 导出反馈数据为 CSV
.mode csv
.output feedbacks.csv
SELECT * FROM Feedback;
.output stdout
```

### Q5：统计数据不准确？
**A：** 可能原因：
1. 数据还在加载中
2. 缓存问题

**解决方法：**
- 刷新页面
- 切换到其他标签再切换回来
- 清除浏览器缓存

## 💡 使用技巧

### 1. 定期查看反馈
建议每天至少查看一次反馈，及时发现和解决问题。

### 2. 关注活跃用户
活跃用户的反馈往往更有价值，优先处理。

### 3. 分析功能使用情况
根据功能排行优化产品，把资源投入到最受欢迎的功能上。

### 4. 监控 AI 性能
如果平均响应时间过长（>30秒），需要优化或更换 API。

### 5. 建立反馈处理流程
1. 每天固定时间查看反馈
2. 分类整理（Bug/建议/其他）
3. 优先级排序
4. 及时回复用户
5. 记录处理结果

## 🚀 快速测试流程

### 测试反馈功能：

1. **提交测试反馈**
   - 退出登录（或使用另一个浏览器）
   - 访问反馈页面
   - 提交一条测试反馈

2. **查看反馈**
   - 使用管理员账号登录
   - 进入管理后台
   - 点击「用户反馈」标签
   - 应该能看到刚才提交的反馈

3. **验证数据统计**
   - 点击「数据统计」标签
   - 查看各项数据是否正常显示
