## 快速诊断命令

### 检查服务器状态
```bash
lsof -ti:3009  # 查看服务器进程
tail -50 /tmp/next-dev.log  # 查看最新日志
```

### 检查数据库
```bash
ls -lh dev.db  # 查看数据库文件大小
sqlite3 dev.db "SELECT COUNT(*) FROM User;"  # 查看用户数
sqlite3 dev.db "SELECT email FROM User;"  # 查看所有用户邮箱
```

### 重启服务器（清除限流）
```bash
lsof -ti:3009 | xargs kill -9
npm run dev
```

### 查看环境配置
```bash
cat .env | grep -v "^#" | grep -v "^$"
```

## 已知问题和解决方案

### 问题1：登录后立即401
**原因**：Cookie 未正确设置或读取
**解决**：已添加 credentials: 'include' 到所有 fetch 请求
**验证**：检查浏览器 Cookies 是否有 userId

### 问题2：触发限流（429错误）
**原因**：短时间内请求过多
**解决**：重启服务器清除内存中的限流记录
**长期方案**：使用 Redis 或调整限流阈值

### 问题3：AI 功能不工作
**原因**：未配置 ANTHROPIC_API_KEY
**解决**：在 .env 中添加 API key 并重启服务器

### 问题4：密码不安全
**原因**：使用 Base64 编码而非加密
**解决**：升级到 bcrypt（需要迁移现有密码）
